🌿 5.1 Git 版本控制完整教程

⏮️

为什么要用Git？

想象你改了一个脚本，发现新版本有bug，但老版本可以用，你怎么回到老版本？
如果没有Git：翻历史备份，也许找不到了。
有了Git：git checkout 秒回任意历史版本，还能看到每次改了什么、谁改的。

Git基础命令（每天都用）

bash
# ===== 初始化与配置 ===== git config --global user.name "党光会" git config --global user.email "your@email.com" git init # 在当前目录初始化Git仓库 git clone URL # 克隆远程仓库到本地 # ===== 查看状态 ===== git status # 查看哪些文件被修改了 git log --oneline --graph # 查看提交历史（图形化） git diff # 查看工作区的具体改动 git diff --staged # 查看暂存区的改动 # ===== 日常提交流程 ===== git add . # 暂存所有变更（常用） git add src/main.py # 只暂存指定文件 git add -p # 交互式选择要暂存的内容 git commit -m "fix: 修复订单金额计算错误" # 提交并写描述 # ===== 分支操作 ===== git branch # 查看所有本地分支 git branch -a # 查看所有分支（含远程） git checkout -b feature/add-login # 创建并切换到新分支 git checkout main # 切换到main分支 git merge feature/add-login # 把feature分支合并到当前分支 git branch -d feature/add-login # 删除已合并的分支 # ===== 远程同步 ===== git push origin main # 推送到远程main分支 git push -u origin feature/new # 推送新分支并设置追踪 git pull # 拉取并合并远程最新代码 git fetch # 只拉取不合并 # ===== 撤销操作 ===== git checkout -- file.py # 撤销工作区的修改（未add） git reset HEAD file.py # 撤销暂存（已add，未commit） git revert HEAD # 撤销最后一次提交（安全） git stash # 临时保存工作区（常用！） git stash pop # 恢复临时保存的工作区

Commit 信息规范（面试加分！）

规范的commit信息格式（Conventional Commits）

格式：类型(范围): 描述

类型含义例子
feat新功能feat(rpa): 新增订单自动审批流程
fix修复Bugfix(etl): 修复金额列空值处理错误
refactor重构代码refactor: 抽取公共的API请求函数
docs文档更新docs: 更新README安装说明
test添加测试test: 添加数据清洗函数单元测试
chore构建/工具chore: 更新Python依赖版本

.gitignore 文件（必须配置！）

有些文件不该提交到Git：密码、API Key、临时文件、大数据文件……用.gitignore告诉Git忽略这些。

.gitignore
# Python相关 __pycache__/ *.pyc *.pyo *.pyd .Python *.egg-info/ dist/ build/ # 环境变量（包含密码！绝对不能提交！） .env .env.local *.env # 数据文件（可能很大） *.xlsx *.csv data/ output/ # 日志文件 logs/ *.log # IDE配置 .vscode/ .idea/ *.swp # 虚拟环境 venv/ .venv/ env/
永远不要把包含密码、API Key的文件提交到Git！一旦推到GitHub，即使删除了也可能被人爬取到。

📋 5.2 专业日志系统设计

1

使用loguru（比标准logging更简单）

python
# 安装：pip install loguru from loguru import logger import sys import os # ===== 配置日志（在程序入口处配置一次）===== os.makedirs("logs", exist_ok=True) # 移除默认handler logger.remove() # 添加控制台输出（带颜色） logger.add(sys.stdout, level="INFO", colorize=True, format="{time:HH:mm:ss} | {level} | {message}") # 添加文件输出（每天一个文件，保留30天） logger.add( "logs/{time:YYYY-MM-DD}.log", rotation="00:00", # 每天午夜切换新文件 retention="30 days", # 保留30天 encoding="utf-8", level="DEBUG", format="{time:YYYY-MM-DD HH:mm:ss} | {level} | {function}:{line} | {message}" ) # ===== 使用日志 ===== logger.debug("调试信息：变量值={}", some_var) logger.info("开始处理第{}批数据，共{}条", batch_num, count) logger.success("✅ 订单{}处理成功", order_id) logger.warning("⚠️ 数据异常：{}", detail) logger.error("❌ 处理失败：{}", error_msg) logger.critical("💀 严重错误，程序即将停止") # 记录异常（带完整traceback） try: risky_operation() except Exception as e: logger.exception("操作失败") # 自动记录完整的异常堆栈 # ===== 结构化日志（便于后续分析）===== # 在消息中加入关键字段，方便grep搜索 logger.info(f"[ORDER] id={order_id} status=success amount={amount} customer={customer}") # grep "ORDER" logs/2024-01-15.log | grep "status=error" 就能快速找到所有失败订单

⏰ 5.3 定时任务详解

1

schedule库（简单场景）

python
# 安装：pip install schedule import schedule import time from loguru import logger def morning_sync(): logger.info("执行早间数据同步...") # 实际业务逻辑 def hourly_check(): logger.info("执行每小时检查...") def weekly_report(): logger.info("生成周报...") # 设置各种定时规则 schedule.every().day.at("08:00").do(morning_sync) # 每天8:00 schedule.every().day.at("18:00").do(morning_sync) # 每天18:00也跑一次 schedule.every(30).minutes.do(hourly_check) # 每30分钟 schedule.every().monday.at("09:00").do(weekly_report) # 每周一9点 logger.info("定时任务已启动，等待调度...") while True: schedule.run_pending() time.sleep(60) # 每分钟检查一次是否有任务需要执行

import requests from datetime import datetime DING_WEBHOOK = "https://oapi.dingtalk.com/robot/send?access_token=你的token" def send_alert(title: str, content: str, is_error: bool = False): """发送钉钉告警""" emoji = "🚨" if is_error else "✅" payload = { "msgtype": "markdown", "markdown": { "title": f"{emoji} {title}", "text": f"## {emoji} {title}\n\n{content}\n\n> 时间：{datetime.now().strftime('%Y-%m-%d %H:%M:%S')}" } } try: resp = requests.post(DING_WEBHOOK, json=payload, timeout=5) if resp.json().get("errcode") == 0: print("✅ 钉钉通知发送成功") else: print(f"❌ 发送失败：{resp.text}") except Exception as e: print(f"❌ 网络错误：{e}") # 使用示例 send_alert("数据同步完成", "本次同步：成功100条，失败0条") send_alert("数据同步异常", "**错误信息：** 数据库连接超时\n**建议：** 检查数据库状态", is_error=True)

完整的异常监控装饰器

python
import functools from loguru import logger def monitor_task(task_name: str): """任务监控装饰器：自动记录日志，出错时发告警""" def decorator(func): @functools.wraps(func) def wrapper(*args, **kwargs): start = datetime.now() logger.info(f"[{task_name}] 开始执行") try: result = func(*args, **kwargs) elapsed = (datetime.now() - start).seconds logger.success(f"[{task_name}] 执行成功，耗时{elapsed}秒") send_alert(f"{task_name}执行成功", f"耗时：{elapsed}秒") return result except Exception as e: logger.exception(f"[{task_name}] 执行失败") send_alert(f"{task_name}执行失败", f"错误：{str(e)}", is_error=True) raise return wrapper return decorator # 使用：在函数上加一行装饰器即可自动监控 @monitor_task("每日数据同步") def daily_sync(): # 正常写业务代码，不需要写任何日志和告警代码 fetch_data() process_data() save_result()

# 安装：pip install python-dotenv from dotenv import load_dotenv import os load_dotenv() # 自动读取.env文件 # 读取配置 DB_HOST = os.getenv("DB_HOST") DB_PASSWORD = os.getenv("DB_PASSWORD") API_KEY = os.getenv("API_KEY") # 读取时可以设置默认值 DEBUG = os.getenv("DEBUG", "false").lower() == "true" # 数据库连接字符串用环境变量拼接 DB_URL = f"mysql+pymysql://{os.getenv('DB_USER')}:{os.getenv('DB_PASSWORD')}@{os.getenv('DB_HOST')}/{os.getenv('DB_NAME')}"
