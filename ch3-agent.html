🧠 3.0 理解AI Agent的本质

🤔

传统程序 vs AI Agent 的本质区别

传统程序：你告诉它"如果A就做B，如果C就做D"，它严格按照规则走，规则没覆盖的情况就出错。
AI Agent：你告诉它"帮我完成这个目标"，它自己思考步骤、选择工具、处理异常，就像一个有判断力的员工。

🔍 大模型（LLM）是AI Agent的"大脑"

大模型厂商特点如何调用
GPT-4oOpenAI综合能力最强，英文尤佳openai.com API
文心一言（ERNIE）百度中文理解好，国内合规qianfan.baidu.com

DeepSeek深度求索价格极低，推理能力强api.deepseek.com
智谱AI（ChatGLM）智谱国内，有免费额度open.bigmodel.cn

💡 学习建议：用国产模型练手

✍️ 3.1 Prompt 工程（提示词设计）

💬

什么是Prompt？为什么重要？

Prompt就是你给大模型的"指令"。同样的模型，写得好的Prompt和写得烂的Prompt，效果天壤之别。
做Agent开发，70%的工作是写好Prompt。

📋 Prompt的三种角色

python
from openai import OpenAI client = OpenAI( api_key="your-api-key", base_url="https://api.deepseek.com" # 用DeepSeek，便宜 ) response = client.chat.completions.create( model="deepseek-chat", messages=[ # system：设定AI的角色和行为规范（整个对话都有效） { "role": "system", "content": """你是一个专业的数据分析助手，擅长分析业务数据。 你的工作原则： 1. 回答要简洁、准确 2. 如果数据不足，明确说明需要什么信息 3. 给出具体的数字和比例 4. 使用中文回答""" }, # user：用户的问题 { "role": "user", "content": "这个月销售额120万，上个月100万，请分析" }, # assistant：AI的回答（用于多轮对话，把历史回答也带上） { "role": "assistant", "content": "本月销售额较上月增长20%（20万元），增速良好..." }, # 用户继续追问 { "role": "user", "content": "如果保持这个增速，3个月后能达到多少？" } ] ) print(response.choices[0].message.content)

🎯 高质量Prompt写法技巧

① 明确角色

不好：「帮我写邮件」
好：「你是一个商务邮件专家，帮我写一封…」

② 给具体背景

不好：「分析这份数据」
好：「这是财务部2024年Q1的报销数据，包含字段…，请分析异常项」

③ 指定输出格式

不好：「提取信息」
好：「请以JSON格式输出，包含字段：name, date, amount」

④ 给出示例

提供一两个input→output例子，AI会更精准地理解你想要什么

💡 实战Prompt案例：数据提取

python
def extract_invoice_info(invoice_text): """用AI从发票文字中提取结构化信息""" prompt = f""" 你是一个发票信息提取专家。请从以下发票文字中提取关键信息。 ## 输出要求 严格按照JSON格式输出，不要有多余文字。 ## 输出格式 {{ "invoice_number": "发票号码", "date": "开票日期（格式YYYY-MM-DD）", "seller": "销售方名称", "buyer": "购买方名称", "amount": 金额数字（不含税，数字类型）, "tax": 税额数字（数字类型）, "total": 价税合计（数字类型）, "items": ["货物或服务名称列表"] }} ## 发票内容 {invoice_text} """ response = client.chat.completions.create( model="deepseek-chat", messages=[{"role": "user", "content": prompt}], temperature=0 # temperature=0 输出更稳定，适合结构化提取 ) import json result_text = response.choices[0].message.content return json.loads(result_text) # 使用示例 invoice_text = """ 增值税专用发票 发票号码：12345678 开票日期：2024年01月15日 销售方：北京某某科技有限公司 购买方：济南半亩花田有限公司 货物：软件服务费 不含税金额：86206.90元 税额（13%）：11206.90元 价税合计：100000.00元 """ info = extract_invoice_info(invoice_text) print(json.dumps(info, ensure_ascii=False, indent=2))

📚 3.3 RAG 知识库（让AI读懂你的文档）

🗄️

RAG = Retrieval-Augmented Generation（检索增强生成）

大模型只知道训练数据里的知识。你公司的内部规定、产品手册、历史数据，AI不知道。
RAG的原理：先把你的文档分片存入向量数据库 → 用户提问时先在文档里搜索相关内容 → 把搜索结果和问题一起发给AI → AI基于你的文档回答。

📄 上传文档
PDF/Word/Excel
→
✂️ 文档分片
→
🗄️ 向量化存储
↑↓
🔍 相似度检索
→
🤖 AI生成回答

在Dify中如何使用RAG？

  1. 左侧导航 → 知识库 → 创建知识库
  2. 上传文档（支持PDF、Word、Excel、TXT等）
  3. 等待文档处理完成（自动切片和向量化）
  4. 在工作流中添加"知识库检索"节点，连接这个知识库
  5. 将检索结果传给LLM节点，让AI参考文档内容回答

⚙️ 3.4 LangChain 代码级 Agent 开发

什么时候用LangChain代码，什么时候用Dify？

Dify适合：标准工作流、不复杂的逻辑、快速上线
LangChain代码适合：复杂逻辑、需要深度定制、集成复杂系统

python
# 安装：pip install langchain langchain-openai from langchain_openai import ChatOpenAI from langchain.agents import AgentExecutor, create_tool_calling_agent from langchain.tools import tool from langchain_core.prompts import ChatPromptTemplate # ===== 1. 定义工具（Agent可以调用的函数）===== @tool def query_order_status(order_id: str) -> str: """查询订单的当前状态。 Args: order_id: 订单号，格式如 ORD-2024-001 """ # 实际上这里会查数据库或调API # 这里用模拟数据 mock_data = { "ORD-001": "已发货，预计明天到达", "ORD-002": "待审批", "ORD-003": "已完成" } return mock_data.get(order_id, f"未找到订单 {order_id}") @tool def get_employee_info(name: str) -> str: """获取员工信息。 Args: name: 员工姓名 """ employees = { "张三": "财务部，职位：会计，工号：FIN001", "李四": "运营部，职位：运营专员，工号：OPS002" } return employees.get(name, f"未找到员工 {name}") # ===== 2. 创建大模型 ===== llm = ChatOpenAI( model="deepseek-chat", base_url="https://api.deepseek.com/v1", api_key="your-deepseek-key", temperature=0 ) # ===== 3. 定义Agent的Prompt ===== prompt = ChatPromptTemplate.from_messages([ ("system", "你是一个智能助手，可以查询订单和员工信息。请用中文回答。"), ("human", "{input}"), ("placeholder", "{agent_scratchpad}") # Agent思考过程的占位符 ]) # ===== 4. 创建并运行Agent ===== tools = [query_order_status, get_employee_info] agent = create_tool_calling_agent(llm, tools, prompt) executor = AgentExecutor(agent=agent, tools=tools, verbose=True) # 提问！Agent会自动决定调用哪些工具 result = executor.invoke({"input": "ORD-001的状态是什么？张三在哪个部门？"}) print(result["output"]) # 输出示例（verbose=True可以看到Agent的思考过程）： # 思考：需要查询订单ORD-001的状态，调用query_order_status("ORD-001") # 观察：已发货，预计明天到达 # 思考：需要查询张三的信息，调用get_employee_info("张三") # 观察：财务部，职位：会计，工号：FIN001 # 最终回答：ORD-001已发货，预计明天到达。张三在财务部，担任会计职位。
